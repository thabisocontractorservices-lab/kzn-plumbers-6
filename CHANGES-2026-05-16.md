# Changes in this bundle (2026-05-16, updated)

Now contains the original SEO indexing fixes plus IndexNow integration (added later same day).

## 1. `app/plumber/[id]/page.tsx` — added `generateMetadata()`

**Why:** Every plumber profile was serving the same default `<title>` and `<meta description>` from `app/layout.tsx`. Google rejected 30 profiles with "Crawled - currently not indexed" because all 1,217 looked like duplicates.

**What it does now:** Generates a unique title (e.g. *"Mannix Plumbers — PIRB-certified plumber in Pinetown | KZN Plumbers"*), a unique meta description (from real `about` if present, else built from area + specialties), canonical URL, Open Graph tags, and Twitter card per profile.

## 2. `components/RegisterWizard.tsx` — added `emailRedirectTo` to `signUp()`

**Why:** Without it, the confirmation link in signup emails relied on whatever the Supabase dashboard's "Site URL" was set to. Brittle.

**What it does now:** Explicitly tells Supabase to send users to `{origin}/auth/callback?next=/dashboard` after they confirm.

## 3. `components/ClaimFlow.tsx` — same `emailRedirectTo` fix

Same reason as #2, applied to the claim-listing signup path.

## 4. `scripts/backfill-plumber-about.sql` — new SQL script

**Why:** Most of the 1,217 plumbers imported from CSV have no `about` description. The profile page renders no "About" panel when blank, leaving body content thin.

**What it does:** Generates 4 different sentence templates rotated by id hash, using real plumber data (name, area, specialties, certifications, emergency flag). Doesn't touch plumbers who already have a real description.

**How to run:** Open Supabase Dashboard → SQL Editor → paste the file contents → Run. The file has a commented-out DRY RUN block at the top — uncomment it first to preview 5 rows before committing.

## 5. `app/api/indexnow/route.ts` + `public/001e7b24e19455300da367300ac77b65.txt` — IndexNow integration

**Why:** Google has 95 sitemap URLs "Discovered - not yet crawled." IndexNow notifies Bing, Yandex, and other engines instantly, which speeds up crawl for ALL of them (including signaling to Google indirectly).

**What it does:**
- Adds an endpoint at `/api/indexnow` that reads all your SEO content pages + plumber profiles from Supabase and POSTs them to IndexNow in one request
- Adds the IndexNow key verification file at `/001e7b24e19455300da367300ac77b65.txt` so engines can verify you own the domain

**How to trigger:** After this deploys, visit `https://www.kznplumbers.co.za/api/indexnow` in your browser. You'll see a JSON response like:

```json
{
  "ok": true,
  "statusCode": 200,
  "submitted": 1770,
  "indexNowResponse": "...",
  ...
}
```

`ok: true` + `statusCode: 200 or 202` = all URLs successfully queued. Bing typically reflects within hours, Yandex similar. Google sees the signals over the following days.

**Optional follow-up:** Set up a Vercel cron job to re-ping IndexNow daily so any newly added plumbers are submitted automatically (use `?secret={CRON_SECRET}` to scope it).

## 6. `app/all-plumbers/page.tsx` + Footer link + sitemap + IndexNow update — HTML sitemap page

**Why:** Google had ~95 plumber profile URLs "Discovered - currently not indexed" because there was no single page where every profile was one click away. The crawler had to paginate through the homepage directory to find them, which Google's crawl budget rarely permits on a new site.

**What it does:**
- New page at `/all-plumbers` lists every verified plumber, grouped by area (Durban, Pinetown, Ballito, etc.), alphabetical within each area
- Each plumber shown as a compact card with PIRB / 24-7 badges, linking directly to `/plumber/[slug]`
- Top of page has a "Jump to area" table-of-contents anchor menu
- Footer link from every page on the site → so Google discovers `/all-plumbers` from anywhere
- Added to sitemap.xml with high priority (0.9, refresh daily)
- Added to IndexNow ping endpoint so Bing/Yandex know about it
- Static generation with ISR (revalidates daily)

**Effect:** Google's crawler now has a single-page route to every plumber profile in one HTTP request. Expected outcome: the "Discovered - not indexed" bucket shrinks meaningfully over 2-4 weeks as Google's crawler systematically picks off profiles from this index.

---

## Deploy order

1. Push these files to GitHub (via web UI or by replacing the repo contents)
2. Vercel auto-deploys within ~2 min
3. Once deployed, run the SQL backfill in Supabase Studio
4. In Search Console, click "Request indexing" on 5-10 of your rejected plumber URLs
5. Wait 2-4 weeks for Google to re-crawl and reflect the changes

## What this doesn't include

- Custom Supabase email templates (paste those manually into Auth → Email Templates)
- DMARC TXT record at your DNS registrar
- Resend domain verification (3 DNS records at your registrar)

Those four steps live outside the codebase — see the project doc in Hyperagent for the exact values.
