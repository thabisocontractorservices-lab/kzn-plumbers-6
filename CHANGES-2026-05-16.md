# Changes in this bundle (2026-05-16)

Three fixes baked in. Two are code, one is a SQL script you run separately.

## 1. `app/plumber/[id]/page.tsx` — added `generateMetadata()`

**Why:** Every plumber profile was serving the same default `<title>` and `<meta description>` from `app/layout.tsx`. Google rejected 30 profiles with "Crawled - currently not indexed" because all 1,217 looked like duplicates.

**What it does now:** Generates a unique title (e.g. *"Mannix Plumbers — PIRB-certified plumber in Pinetown | KZN Plumbers"*), a unique meta description (from real `about` if present, else built from area + specialties), canonical URL, Open Graph tags, and Twitter card per profile.

## 2. `components/RegisterWizard.tsx` — added `emailRedirectTo` to `signUp()`

**Why:** Without it, the confirmation link in signup emails relied on whatever the Supabase dashboard's "Site URL" was set to. Brittle.

**What it does now:** Explicitly tells Supabase to send users to `{origin}/auth/callback?next=/dashboard` after they confirm.

## 3. `components/ClaimFlow.tsx` — same `emailRedirectTo` fix

Same reason as #2, applied to the claim-listing signup path.

## 4. `scripts/backfill-plumber-about.sql` — new SQL script

**Why:** Most of the 1,217 plumbers imported from CSV have no `about` description. The profile page renders no "About" panel when blank, leaving body content thin.

**What it does:** Generates 4 different sentence templates rotated by id hash, using real plumber data (name, area, specialties, certifications, emergency flag). Doesn't touch plumbers who already have a real description.

**How to run:** Open Supabase Dashboard → SQL Editor → paste the file contents → Run. The file has a commented-out DRY RUN block at the top — uncomment it first to preview 5 rows before committing.

---

## Deploy order

1. Push these files to GitHub (via web UI or by replacing the repo contents)
2. Vercel auto-deploys within ~2 min
3. Once deployed, run the SQL backfill in Supabase Studio
4. In Search Console, click "Request indexing" on 5-10 of your rejected plumber URLs
5. Wait 2-4 weeks for Google to re-crawl and reflect the changes

## What this doesn't include

- Custom Supabase email templates (paste those manually into Auth → Email Templates)
- DMARC TXT record at your DNS registrar
- Resend domain verification (3 DNS records at your registrar)

Those four steps live outside the codebase — see the project doc in Hyperagent for the exact values.
